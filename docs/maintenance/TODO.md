## 📚 第一章：專案歷史與背景 (History & Context)

### 1.1 初始狀態 (The Beginning)
本站點為 MDM 技術支援文件庫，擁有 **繁體中文 (zh)** 與 **英文 (en)** 雙語版本。
*   **問題源頭**：中文版內容經過嚴格審核，格式統一且標準。英文版則由不同人員或時期翻譯，導致格式混亂（例如：列表符號混用 `*` 與 `-`、標題間距不一、術語過時）。
*   **觸發事件**：為了提升閱讀體驗並方便未來自動化工具（如 Diff）進行雙語對照，我們啟動了 **「雙語同步與格式標準化」** 專案。

### 1.2 執行歷程 (Chronology)
1.  **第一階段：排版修正 (Layout Fixes)**
    *   修正了 Markdown 語法錯誤（如表格跑版、連結失效）。
    *   處理了 `digital-learning` 章節的部分檔案。
2.  **第二階段：帳號章節同步 (Account Sync)**
    *   針對 `account` (acc-1 ~ acc-26) 進行了地毯式修正。
    *   **重大決策**：確立「中文版為黃金標準 (Golden Master)」，英文版必須在「原始碼層級」上與中文版行數、結構完全對應。
3.  **第三階段：輔助工具開發 (Tooling)**
    *   為了提升效率，我們開發了 PowerShell 腳本，將零散的 `.md` 檔案合併為 **Bundle (合併檔)**。
    *   **目的**：讓維護者能在一个檔案內看完所有章節，不用反覆開關檔案。
4.  **第四階段：標準化定案 (Standardization)**
    *   **決策 A**：全面棄用星號 `*` 作為列表符號，改為減號 `-`（與中文版一致）。
    *   **決策 B**：強制標題 (`##`, `###`) 上方必須保留一行空白行。
    *   **相應調整**：修改 `.markdownlint.json` 與 CSS 樣式表以配合此標準。

## 🛠️ 第二章：技術架構與設定 (Technical Configuration)

為了支持上述決策，專案設定檔進行了以下調整，請勿隨意更動：

### 2.1 Lint 規則 (`.markdownlint.json`)
目前處於 **「過渡遷移期 (Migration Phase)」**，設定如下：

*   **`MD004` (列表樣式)**：暫設為 `false`。
    *   *原因*：雖然目標是全改為 `-`，但因為檔案眾多，若強制檢查會導致 CI/CD 全面報錯。我們允許在過渡期混用，待全部修正完畢後再設為 `"dash"`。
*   **`MD022` & `MD032` (空白行檢查)**：設為 `true`。
    *   *原因*：標題與列表周圍的空白行嚴重影響閱讀與渲染，必須強制執行。

### 2.2 視覺樣式 (`docs/.vitepress/theme/markdown-styles.css`)
我們更新了 CSS 以優化長文閱讀體驗：
*   **Typography**：針對無序列表 (`ul`) 強制定義了樣式，確保無論原始碼是用 `*` 還是 `-`，在網頁前端看起來都是一致的圓點 `•`，避免過渡期的視覺混亂。
*   **H2 標題**：增加了左側裝飾線與間距，與 Markdown 原始碼中的「上方空行」邏輯呼應。

## 📝 第五章：標準作業程序 (SOP)

### 5.1 雙螢幕/分割視窗作業法 (推薦)
1.  **開啟對照檔案**：
    *   左側視窗：`docs/maintenance/bundles/zh_[類別]_bundle.md`
    *   右側視窗：`docs/maintenance/bundles/en_[類別]_bundle.md`
2.  **視覺掃描**：
    *   同步捲動，檢查段落長度是否異常（可能代表缺漏）。
    *   檢查列表符號：看到 `*` 就必須修正。
3.  **修正原始檔**：
    *   **注意**：Bundle 檔僅供檢視。
    *   發現錯誤時，請根據 Bundle 分隔線上的檔名 (`--- File: xxx.md ---`)，去開啟真正的原始檔進行編輯。

### 5.2 術語校對準則 (Terminology Guide)
在進行格式修正時，若看到舊術語，請順手修正：

| 舊術語 (Old) | 新術語 (New/Target) | 備註 |
| :--- | :--- | :--- |
| **`Managed Apple ID`** | **`Managed Apple Account`** | 2024年底更名 |
| **`Apple ID`** | **`Apple Account`** | 同上 |
| `DEP` / `Device Enrollment Program` | `ADE` / `Automated Device Enrollment` | 請使用現代術語 |
| `VPP` / `Volume Purchase Program` | `Apps and Books` | 在 ASM 介面中已更名，但 VPP Token 仍通用 |
